# Memory Templates

The canonical skeletons that `scripts/seed-memory.sh` emits for a fresh repo,
plus example content showing what each file looks like after a real bootstrap.

`bootstrap` writes these on first visit. `wrap-up` appends and revises them
every session. The project `.claude/CLAUDE.md` loads MEMORY.md, architecture.md,
todo.md, lessons.md, and GRAPH_REPORT.md via `@` on every session start —
history.md is intentionally excluded (append-only, grows large).

## Contents

- File-by-file purpose
- MEMORY.md — index
- architecture.md — stack, modules, conventions
- todo.md — Active / Backlog / Done
- lessons.md — Patterns / Anti-patterns / Wins
- history.md — append-only session log
- Validation checklist
- How wrap-up mutates these files

## File-by-file purpose

| File | What it holds | Auto-loaded | Rewritten by bootstrap |
|---|---|---|---|
| `MEMORY.md` | Pure index of the other 4 + graph report | yes | every run (idempotent) |
| `architecture.md` | Stack, entry points, conventions | yes | first run only |
| `todo.md` | Active / Backlog / Done lists | yes | first run only |
| `lessons.md` | Patterns / Anti-patterns / Wins | yes | first run only |
| `history.md` | Append-only session log | **no** | first run only (seeded entry) |

Rule: `seed-memory.sh` preserves existing content on re-run. Only `MEMORY.md`
is always overwritten because it derives mechanically from the layout.

## MEMORY.md — index

Pure pointer file. Always regenerated by `seed-memory.sh`. No narrative, no
state — five Markdown links plus a footer comment.

```markdown
# <REPO_NAME> Memory Index

- [Architecture](~/.claude/projects/<REPO_NAME>/memory/architecture.md) — tech stack, modules, entry points, conventions
- [Todo](~/.claude/projects/<REPO_NAME>/memory/todo.md) — active tasks and backlog
- [Lessons](~/.claude/projects/<REPO_NAME>/memory/lessons.md) — patterns and anti-patterns
- [History](~/.claude/projects/<REPO_NAME>/memory/history.md) — session history (not auto-loaded)
- [Graph Report](~/.claude/projects/<REPO_NAME>/graphs/GRAPH_REPORT.md) — codebase knowledge graph

<!-- Auto-loaded via @ references in <repo>/.claude/CLAUDE.md on every session start. -->
```

## architecture.md — stack, modules, conventions

Written once by bootstrap, extended by wrap-up when structure drifts. Source
material comes from `$GRAPHS_DIR/GRAPH_REPORT.md`.

Target: 100–200 lines. Sections are fixed.

```markdown
# <REPO_NAME> — Architecture

## Tech Stack
| Layer | Tech |
|---|---|
| Orchestration | Airflow 2.9 |
| Compute       | PySpark 3.4 |
| Warehouse     | Snowflake   |
| CI            | GitHub Actions + OIDC |

## Entry Points
- `src/main.py` — CLI application entrypoint
- `dags/etl_daily.py` — daily ETL DAG
- `jobs/spark_aggregate.py` — nightly Spark job

## Key Modules
- `src/pipeline/extract.py` — source adapters (in_degree: 12)
- `src/db/session.py` — connection pool (in_degree: 19)
- `src/utils/config.py` — env + secrets (in_degree: 28)

## Data Flow
- Source systems → Kafka topics → PySpark micro-batch → Snowflake staging → dbt transforms → marts
- Airflow orchestrates the dbt run nightly; observability via New Relic

## Conventions
- Config: env vars via python-dotenv; secrets in Vault
- Tests: pytest in `tests/`, fixtures in `conftest.py`; `@pytest.mark.integration` for DB tests
- Naming: snake_case modules, PascalCase classes, SCREAMING_SNAKE constants
- SQL: dbt only; no raw SQL outside `sql/` dir

## Known Risks
- `legacy_ingest.py` has no tests, called by 3 DAGs — schedule rewrite
- Snowflake warehouse sizing hardcoded — should come from env
```

## todo.md — Active / Backlog / Done

Working queue. Wrap-up promotes items between sections. Keep Active ≤3 to
enforce focus.

```markdown
# <REPO_NAME> — Todo

## Active
- [ ] Backfill April 2026 sales data after schema migration
- [ ] Wire New Relic alert for DAG failures (terraform module)

## Backlog
- [ ] Replace `legacy_ingest.py` with dbt snapshot pattern
- [ ] Add retry policy to Snowflake writes
- [ ] Document on-call runbook for Kafka consumer lag

## Done
- 2026-05-01 — Ran /bootstrap; seeded memory and graph.
- 2026-04-30 — Migrated Airflow 2.7 → 2.9; all DAGs green.
- 2026-04-28 — Added dbt tests for `mart_revenue_daily`.
```

## lessons.md — Patterns / Anti-patterns / Wins

Accumulates across sessions. Wrap-up captures new entries from user
corrections and completed work.

```markdown
# <REPO_NAME> — Lessons

## Patterns
- **Use lazy imports inside Airflow task functions** — module-level imports break the scheduler.
- **Parameterize every SQL query** — f-strings are a security review blocker.
- **Every Spark job writes to `_SUCCESS` marker before mart table** — idempotency anchor.

## Anti-patterns
- **No large XCom payloads** — pass S3/GCS paths instead.
- **No `datetime.utcnow()`** — use `datetime.now(timezone.utc)`.
- **No print() in services** — structlog with `.bind()` is the standard.

## Wins
- Switched daily reconciliation to incremental dbt model — 20× faster, same correctness.
- Salting on skewed join keys cut p95 job runtime from 48m → 12m.
```

## history.md — append-only session log

Not loaded into context by default. Only opened when the user asks for recall
("what did we do last week?"). Keeps grow-forever narrative out of the active
context window.

```markdown
# <REPO_NAME> — Session History

<!-- Append-only. Newest at top. -->

## 2026-05-01 — Bootstrap
- Ran /bootstrap; seeded memory + graph.
- Stack: Python 3.12, Airflow 2.9, PySpark 3.4, Snowflake, dbt 1.7.
- 142 Python files, 30 SQL models, 23 YAML configs.

## 2026-05-01 — Alert wiring session
- Added New Relic NRQL condition for DAG task failures.
- Fixed flaky integration test in `test_snowflake_load.py`.
- Promoted 2 items from Backlog → Active.
```

## Validation checklist

Run after `seed-memory.sh` finishes:

- [ ] `test -f $MEMORY_DIR/MEMORY.md` and contains 5 links.
- [ ] `test -f $MEMORY_DIR/architecture.md` with the 6 canonical section headers.
- [ ] `test -f $MEMORY_DIR/todo.md` with `## Active`, `## Backlog`, `## Done`.
- [ ] `test -f $MEMORY_DIR/lessons.md` with `## Patterns`, `## Anti-patterns`, `## Wins`.
- [ ] `test -f $MEMORY_DIR/history.md` with a dated bootstrap entry.
- [ ] `grep -c '^@~/.claude/projects' <repo>/.claude/CLAUDE.md` returns `5`.
- [ ] `.claude/` appears in `<repo>/.gitignore`.

If any check fails, re-run the offending script with the correct args. Never
edit MEMORY.md by hand — regenerate.

## How wrap-up mutates these files

| File | Wrap-up behavior |
|---|---|
| `MEMORY.md` | Never touched (bootstrap owns it). |
| `architecture.md` | Updated when graph refreshes; otherwise left alone. |
| `todo.md` | Promotes completed items to `## Done`; appends new backlog items. |
| `lessons.md` | Appends new `## Patterns` / `## Anti-patterns` from user corrections. |
| `history.md` | Prepends a dated session summary. Always. |

`wrap-up` never overwrites whole files. If a file gets out of shape, delete it
and re-run `bootstrap` with confirmation.
